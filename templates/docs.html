Developer API Documentation

Integrate the power of Sparkling Studio Pro into your own applications. Our API uses a Task Polling Architecture to ensure absolute stability and bypass serverless execution timeouts (like Vercel). The Gateway securely handles all Heavy GPU routing and authorization invisibly.

1. Endpoints

POST https://api.example.com/api/generate

Initiates a new image generation task. Returns a unique task_id immediately.

Request Body (JSON)

ParameterTypeDefaultDescription
promptstringRequiredDescribe the scene (e.g., "Lord Shiva meditating in deep space").
user_negativestring""Elements to avoid in the image.
style_namestring"Cinematic""Cinematic", "Realistic", "Anime", "Cosmic", "Oil Painting", "No Style"
ratiostring"1:1""1:1", "16:9", "9:16", or "Custom"
GET https://api.example.com/api/status/{task_id}

Poll this endpoint every 1-2 seconds using the task_id received from the generate endpoint to get the image status.

Response Statuses

  • queued: Waiting for engine availability.
  • processing: Currently generating via AI models.
  • completed: Generation done. Response includes image (Base64) and seed.
  • failed: Task encountered an error. Response includes error message.

2. Code Examples

Choose your preferred language. The Base URL is dynamically updated to match your current environment.

Python (Requests) 
import requests
import time

BASE_URL = "https://api.example.com"

def generate_image():
    # 1. Start the Task
    payload = {
        "prompt": "Lord Shiva meditating in deep space, glowing cosmic energy",
        "style_name": "Cosmic",
        "ratio": "16:9"
    }
    
    response = requests.post(f"{BASE_URL}/api/generate", json=payload)
    data = response.json()
    
    if data.get("cached"):
        print("Magic Cache Hit! Image retrieved instantly.")
        return data["image"]
        
    task_id = data["task_id"]
    print(f"Task Started: {task_id}. Polling for completion...")
    
    # 2. Poll for Status
    while True:
        status_res = requests.get(f"{BASE_URL}/api/status/{task_id}").json()
        status = status_res.get("status")
        
        if status == "completed":
            print("Generation Successful!")
            return status_res["image"]
        elif status == "failed":
            raise Exception(status_res.get("error", "Unknown generation error"))
            
        time.sleep(1.5)

image_base64 = generate_image()
const BASE_URL = "https://api.example.com";

async function generateImage() {
    // 1. Start the Task
    const payload = {
        prompt: "Lord Shiva meditating in deep space, glowing cosmic energy",
        style_name: "Cosmic",
        ratio: "16:9"
    };

    let res = await fetch(`${BASE_URL}/api/generate`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(payload)
    });
    let data = await res.json();

    if (data.cached) {
        console.log("Magic Cache Hit! Image retrieved instantly.");
        return data.image;
    }

    const taskId = data.task_id;
    console.log(`Task Started: ${taskId}. Polling...`);

    // 2. Poll for Status
    while (true) {
        let statusRes = await fetch(`${BASE_URL}/api/status/${taskId}`);
        let statusData = await statusRes.json();

        if (statusData.status === 'completed') {
            console.log("Generation Successful!");
            return statusData.image;
        } else if (statusData.status === 'failed') {
            throw new Error(statusData.error);
        }

        await new Promise(resolve => setTimeout(resolve, 1500));
    }
}

generateImage();

Error Handling & Responses

Our API uses standard HTTP status codes combined with JSON error descriptions.

  • 200 OK: Request succeeded. Always check the JSON status field.
  • 404 Not Found: The task_id does not exist or has expired from the live queue.
  • 500 Internal Error: The Gateway could not communicate with the Private Heavy-Logic Backend.
